| Command       | Purpose                                                                                              | When to Use                                                                       |
| ------------- | ---------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------- |
| **git clone** | Creates a complete copy of a remote repository on your local computer.                               | Use when you are downloading a repository for the first time.                     |
| **git fetch** | Downloads the latest changes from the remote repository without merging them into your local branch. | Use when you want to check for updates before merging.                            |
| **git pull**  | Downloads the latest changes and automatically merges them into your current branch.                 | Use when you want to update your local repository with the latest remote changes. |


Q10. What is a .gitignore file? (2 Marks)
A .gitignore file tells Git which files and folders should not be tracked or uploaded to the repository. It is used to exclude temporary files, compiled files, build folders, and IDE settings.

Arduino Project .gitignore:
# Compiled output files
*.hex
*.elf
# Build folder
build/
# IDE configuration
.vscode/
# OS-specific files
.DS_Store
Thumbs.db
This .gitignore ensures that only the source code is stored in the Git repository while unnecessary generated files are ignored.
